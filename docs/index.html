FluentUI

Librería C++ de UI inmediata con estética Fluent Design 2, backend OpenGL + SDL3, docking, multi-viewport y layouts responsivos.

Introducción

FluentUI es una librería de interfaz gráfica al estilo immediate-mode (similar a Dear ImGui) pero con la apariencia de Microsoft Fluent Design 2. Cada frame se reconstruye desde cero: las funciones de widget devuelven el estado de interacción y el llamador es responsable de mantener los datos.

Características principales

  • Más de 40 widgets: botones, sliders, drag editors, combos, listas, árboles, tablas, plots, pickers de fecha/hora.
  • Sistema de docking completo con split + tab + paneles flotantes.
  • Multi-viewport: paneles que se desprenden a ventanas OS independientes.
  • Layouts responsivos tipo CSS: Fill, Fixed, Auto.
  • Theming por defecto claro/oscuro + pila de overrides.
  • Soporte DPI por monitor, IME, drag & drop tipado, atajos de teclado.
  • Renderer batched OpenGL con MSDF para texto.

Dos APIs equivalentes

Cada widget está disponible en dos formas:

  • Funciones libres al estilo ImGui (FluentUI::Button(...)): control fino y mínima sobrecarga.
  • UIBuilder con lambdas (ui.button(...)): API fluida ideal para layouts anidados.

Puedes mezclar ambas en el mismo frame.

Instalación

FluentUI se distribuye como código fuente. Requisitos:

  • Compilador C++20 (MSVC 2022, GCC 11+, Clang 14+).
  • CMake 3.20+.
  • SDL3 (vendored o por vcpkg).
  • OpenGL 3.3+ (glad incluido en external/).

Build con CMake

git clone https://github.com/dabo96/WinUI3.git fluentui
cd fluentui
cmake -S . -B build
cmake --build build --config Release

Integrar en tu proyecto

add_subdirectory(third_party/fluentui)
target_link_libraries(MyApp PRIVATE FluentUI)

Header umbrella

Para incluir todo de una sola vez:

#include <FluentUI/API.h>

Tu primera ventana

El esqueleto mínimo de una app FluentUI:

#include <FluentUI/API.h>
using namespace FluentUI;

int main() {
    AppConfig cfg;
    cfg.width = 1280;
    cfg.height = 720;
    cfg.darkMode = true;

    FluentApp app("Hola FluentUI", cfg);

    app.root([](UIBuilder& ui) {
        ui.label("Hola, mundo!", TypographyStyle::Title);
        ui.button("Cerrar", {120, 32});
    });

    return app.run();
}

FluentApp crea la ventana SDL, el contexto OpenGL, el contexto de UI y el bucle principal. app.root(...) registra la lambda que se ejecuta cada frame para construir la UI.

Modos de ejecución: app.run() entra en un loop bloqueante. Si tu motor ya tiene su propio loop (por ejemplo un editor 3D), usa la API processEvent / beginFrame / endFrame con un SDL_Window y contexto GL externos.

Integración con motor existente

SDL_Window* win = /* tu ventana */;
SDL_GLContext gl = /* tu contexto */;

// Constructor para ventana externa: NO recibe título —
// la ventana ya existe y FluentApp no la gestiona.
FluentApp app(win, gl, /*darkMode*/true, /*enableDPI*/true);

while (running) {
    SDL_Event e;
    while (SDL_PollEvent(&e)) {
        // tu motor procesa primero...
        app.processEvent(e);
    }
    float dt = computeDt();

    // 1) renderiza tu escena 3D
    renderScene();

    // 2) UI encima
    app.beginFrame(dt);
    // (root builder se ejecuta automáticamente)
    app.endFrame();

    SDL_GL_SwapWindow(win);
}

Conceptos clave

Inmediato vs retenido

La UI no se "construye una vez" — se redibuja completa cada frame. Cada widget recibe punteros a tu propio estado y devuelve true cuando ha cambiado.

static bool checked = false;
if (Checkbox("Activar opción", &checked)) {
    // se ejecuta solo en el frame que cambió
    log("Toggled to %d", checked);
}

IDs y colisiones

FluentUI usa el label como identidad. Si tienes dos botones con el mismo texto, dales un sufijo único:

Button("Eliminar##fila1");
Button("Eliminar##fila2");

El texto tras ## se ignora visualmente pero entra en el ID.

Posicionamiento absoluto

Cada widget puede recibir un std::optional<Vec2> para fijar su posición; si lo omites se coloca según el cursor del layout actual.

Layouts vertical y horizontal

Los layouts gestionan posición y tamaño de los hijos. El cursor avanza verticalmente por defecto.

void BeginVertical(spacing=-1, size=auto, padding=auto)
void BeginHorizontal(spacing=-1, size=auto, padding=auto)
// Estilo función libre
BeginHorizontal(8.0f);
  Button("Guardar");
  Button("Cancelar");
EndHorizontal();

// Estilo builder con lambdas
ui.horizontal([&](UIBuilder& r) {
    r.button("Guardar");
    r.button("Cancelar");
});

Spacing y SameLine

Spacing(12.0f);            // 12 px verticales
Button("A");
SameLine(10.0f);           // misma línea, 10 px de separación

El enum ButtonSize aplica presets Fluent 2 que sobrescriben padding/fontSize del estilo actual:

VarianteAltoPadding (X×Y)Tamaño de fuente
Small~24 px8 × 312 (Caption)
Medium (default)~32 px12 × 514 (Body)
Large~40 px16 × 816

Pasa label="" a Button(label, iconCp) para obtener un botón cuadrado solo-icono. IconButton es el atajo idiomático equivalente. El iconCodepoint resuelve a un glyph de la fuente secundaria (ver sección Iconos); 0 = sin icono.

TextInput con callbacks

Para autocompletado, validación o historial:

static std::string cmd;
uint32_t mask = TextInputCallbackType::Completion |
                TextInputCallbackType::CharFilter;

TextInput("Comando", &cmd, 320, false, std::nullopt, "> …", 0,
    [](TextInputCallbackData& d) {
        if (d.type == TextInputCallbackType::CharFilter) {
            if (!isalnum((int)d.charInput) && d.charInput != ' ')
                d.charInput = 0;          // rechaza
        } else if (d.type == TextInputCallbackType::Completion) {
            *d.buffer += "_completed";    // expande con Tab
            d.cursorPos = d.buffer->size();
        }
    }, mask);

ComboBox / ComboBoxNoLabel

bool ComboBox(label, *currentItem, items, width=200, pos=auto)
bool ComboBox(label, *currentItem, vector<pair<string,uint32_t>> items, width=200, pos=auto)
bool ComboBoxNoLabel(id, *currentItem, items, width=200, pos=auto)
bool ComboBoxNoLabel(id, *currentItem, vector<pair<string,uint32_t>> items, width=200, pos=auto)
std::vector<std::string> engines = {"OpenGL","Vulkan","DX11","Metal"};
static int sel = 0;
ComboBox("Backend", &sel, engines);

El overload con vector<pair<string,uint32_t>> dibuja un icono (de la fuente secundaria — ver sección Iconos) a la izquierda del texto seleccionado y de cada fila del dropdown. Cuando un item tiene icono se sustituye el "selection dot" por el icono coloreado con el accent.

static int meshIdx = 0;
ComboBox("Mesh", &meshIdx,
    std::vector<std::pair<std::string, uint32_t>>{
        {"PlayerBody.fbx", Icons::Box},
        {"Enemy.fbx",      Icons::Box},
        {"Crate.fbx",      Icons::Package},
    }, 200.0f);

ComboBoxNoLabel tiene exactamente el mismo comportamiento pero no dibuja el header/label encima del campo, ideal para toolbars o property grids donde el contexto ya está claro. El primer parámetro string se usa únicamente como id único del widget (no se muestra), por lo que debe ser distinto entre combos en el mismo scope.

static int coord = 0;
SetNextConstraints(FixedSize(90.0f, 26.0f));
ComboBoxNoLabel("coord", &coord, std::vector<std::string>{"Local","World"});

ComboBoxSearchable

Idéntica firma; cuando se abre muestra una caja de filtro case-insensitive. También dispone del overload con vector<pair> para mostrar iconos por item.

ComboBoxSearchable("País", &countryIdx, allCountries, 280);

DragFloat / DragInt / DragFloat3

Editores numéricos clic-arrastrar (al estilo Blender). Doble-click activa edición por teclado. Mantén Shift para movimiento fino.

bool DragFloat(label, *value, speed=1, min=0, max=0, format="%.2f", pos=auto)
bool DragInt(label, *value, speed=1, min=0, max=0, pos=auto)
bool DragFloat3(label, values[3], speed=1, min=0, max=0, format="%.2f", pos=auto)
static float fov = 60.0f;
DragFloat("FOV", &fov, 0.5f, 10.0f, 170.0f, "%.1f°");

static float pos[3] = {0,0,0};
DragFloat3("Posición", pos, 0.05f);  // X rojo, Y verde, Z azul

Tamaño personalizado: por defecto DragFloat mide 250 px (100 px si el label está vacío) y reserva un 40 % del ancho para dibujar el label. DragFloat3 mide 400 px (300 px sin label). Para fijar exactamente ancho y alto usa SetNextConstraints con FixedSize(w, h) justo antes de la llamada — el campo respetará ambas dimensiones (incluyendo la altura, útil para toolbars compactos):

static float gridSize = 1.0f;
SetNextConstraints(FixedSize(80.0f, 26.0f));
DragFloat("", &gridSize, 0.1f, 0.0f, 100.0f);

static float pos[3] = {0,0,0};
SetNextConstraints(FixedSize(220.0f, 26.0f));
DragFloat3("", pos, 0.05f);

Cuando el label está vacío el slot del header se colapsa a 0, así el campo ocupa todo el ancho indicado. También funcionan FillWidth() / FillSize() si quieres que el widget se estire al contenedor.

Panel

Panel arrastrable con cabecera y contenido contraíble. La sobrecarga con iconCodepoint dibuja un icono a la izquierda del título.

ui.panel("Inspector", [&](UIBuilder& p) {
    p.label("Transform", TypographyStyle::BodyStrong);
    p.dragFloat3("Position", entity.pos);
    p.dragFloat3("Rotation", entity.rot);
    p.dragFloat3("Scale",    entity.scale);
}, /*size*/{320, 0});

\ Variante con icono en el header
ui.panel("Inspector", Icons::Settings, [&](UIBuilder& p) {

El overload extenso permite además: reserveLayoutSpace=false para paneles flotantes que no consumen espacio del layout padre, useAcrylic para forzar el efecto Mica/Acrylic, acrylicOpacity (0-1) y maxHeight para activar scroll interno automáticamente cuando el contenido excede esa altura.

CollapsingHeader

Cabecera plegable estilo Inspector (Unity / Unreal). No requiere EndCollapsingHeader: los widgets que siguen se auto-indentan mientras el header esté expandido. El indent se reinicia con el siguiente CollapsingHeader del mismo nivel o al cerrar el layout padre.

bool CollapsingHeader(label, *open=nullptr, iconCodepoint=0, pos=auto)
// Inspector típico con varias secciones
if (CollapsingHeader("Transform", nullptr, Icons::Move)) {
    DragFloat3("Position", entity.pos);
    DragFloat3("Rotation", entity.rot);
    DragFloat3("Scale",    entity.scale);
}

if (CollapsingHeader("Material", nullptr, Icons::Paintbrush)) {
    ColorPicker("Albedo", &mat.albedo);
    SliderFloat("Metallic",  &mat.metallic,  0.0f, 1.0f);
    SliderFloat("Roughness", &mat.roughness, 0.0f, 1.0f);
}

// Estado externo (recuerda si está abierto/cerrado entre frames)
static bool advancedOpen = false;
if (CollapsingHeader("Advanced", &advancedOpen)) {
    Checkbox("Wireframe", &wireframe);
}

Table / DataGrid

Tabla con sort, redimensionado, scroll vertical y horizontal, columnas congeladas y multi-selección. Cada TableColumn tiene un campo opcional iconCodepoint que dibuja un icono a la izquierda del header.

static std::vector<TableColumn> cols = {
    {"Nombre", 160, 40.0f, true, Icons::Folder},
    {"Tipo",   100}, {"Tamaño", 90}, {"Modificado", 140}
};
static TableState state;
state.frozenColumns = 1;   // congela "Nombre"

if (BeginTable("files", cols, (int)files.size(), {0, 240}, &state)) {
    for (int row = 0; row < (int)files.size(); ++row) {
        TableNextRow();
        TableRowSelectable(row);     // soporta Ctrl/Shift

        TableSetCell(0); Label(files[row].name);
        TableSetCell(1); Label(files[row].type);
        TableSetCell(2); Label(formatBytes(files[row].size));
        TableSetCell(3); Label(files[row].mtime);
    }
    EndTable();
}
Si state.sortColumn != -1 ordena tu vector externamente antes de pasar files a la tabla. La tabla solo indica la columna y dirección — no toca tus datos.

Iconos zero-setup

FluentUI viene con la fuente Lucide preempaquetada y la carga automáticamente al construir FluentApp. No necesitas tocar LoadIconFont: simplemente pasa cualquier valor de Icons::* a un widget icon-aware.

#include <FluentUI/API.h>
using namespace FluentUI;

if (Button("Guardar", Icons::Save)) save();

SegmentedControl("tool", {
    {"", Icons::Pointer}, {"", Icons::Move},
    {"", Icons::Rotate},  {"", Icons::Scale},
}, &tool);

Icons::Save, Icons::Pointer, etc. son entradas de un enum Lucide : uint32_t generado a partir del .ttf real, así que se convierten implícitamente al codepoint que esperan los overloads de los widgets. El catálogo completo (~1500 iconos) vive en include/UI/Icons.h.

Cómo encuentra la fuente

Al construir un FluentApp, la librería busca lucide.ttf en este orden:

  1. AppConfig::iconFontPath si lo proporcionas explícitamente.
  2. <directorio del ejecutable>/assets/fonts/lucide.ttf (ruta canónica vendorizada).
  3. <cwd>/assets/fonts/lucide.ttf.
  4. $FLUENTUI_ASSETS_DIR/fonts/lucide.ttf.

Si nada existe, la app sigue corriendo (los iconos quedan invisibles, no crashea) y se loguea un warning. CMake copia la fuente al directorio del binario automáticamente, así que en builds estándar nunca tienes que preocuparte.

Override avanzado (sustituir la fuente)

Si quieres usar otra fuente (Font Awesome, Fluent UI System Icons, IcoMoon custom, etc.):

AppConfig cfg;
cfg.iconFontPath = "C:/assets/fa-solid.ttf";
cfg.iconFontSize = 18;
FluentApp app("MyApp", cfg);

\ Tus codepoints van directos como uint32_t (sin Icons::):
if (Button("Save", 0xF0C7 /* fa-save */)) save();

El catálogo Icons::* solo es válido para el .ttf de Lucide vendorizado — si lo reemplazas, define tus propios constantes o regenera el header con tools/gen_icons.py apuntando a tu nueva fuente.

Cada FluentApp contiene un ShortcutRegistry. Los atajos se procesan automáticamente cada frame justo después del input.

Undo / Redo

app.undoStack implementa el patrón Command con histórico, agrupación y un helper para valores triviales.

undoStack.Execute(Command{ description, execute, undo });
undoStack.Undo() / Redo() / CanUndo() / CanRedo()
undoStack.BeginGroup(desc) / EndGroup()
MakeValueCommand<T>(desc, *target, newValue) -> Command
// Mutación atómica con undo automático
float oldFov = camera.fov;
if (DragFloat("FOV", &camera.fov, 0.5f, 10.0f, 170.0f)) {
    app.undoStack.Execute({
        "Cambiar FOV",
        [&, newF = camera.fov]{ camera.fov = newF; },
        [&, oldF = oldFov]    { camera.fov = oldF; }
    });
}

// Helper para POD: misma idea pero en una línea
app.undoStack.Execute(MakeValueCommand("Activar wireframe", &wireframe, true));

// Agrupar varias mutaciones bajo un único paso de undo
app.undoStack.BeginGroup("Mover selección");
    for (auto& e : selected) {
        Vec3 oldPos = e.pos;
        e.pos += offset;
        app.undoStack.AddToGroup({
            "", [&, p=e.pos]{ e.pos=p; }, [&, p=oldPos]{ e.pos=p; }
        });
    }
app.undoStack.EndGroup();

File Dialogs nativos

Diálogos OS nativos (no son ventanas FluentUI). Son asíncronos: no bloquean el frame. El callback puede dispararse en otro hilo.

    };

El payload se entrega por copia. Para tipos POD usa el template SetPayload<T> (requiere std::is_trivially_copyable); para rutas u otras cadenas usa el overload de std::string.

// Bezier (Phase 5) — útil para conexiones tipo node graph
r.DrawBezier({100,100},{200,50},{300,150},{400,100},
             FluentColors::Accent, /*thickness*/2.0f, /*segments*/40);

// Gradient

    {50, 400}, {300, 60},
    Color(1,0,0,1), Color(1,1,0,1),
    Color(0,0,1,1), Color(0,1,0,1));

r.DrawRectAcrylic({0,0}, {320, 720}, Color(0.1f,0.1f,0.1f,1),
                  /*radius*/0, /*opacity*/0.85f, /*blur*/12.0f);

            /*uv0*/{0,0}, /*uv1*/{1,1});

DPI & Responsive

FluentUI detecta la escala del monitor automáticamente. Para escalar tus propios valores en píxeles:

float padding = Scaled(8.0f);   // 8 px @ 100%, 16 @ 200%
Vec2 size = { Scaled(120.0f), Scaled(32.0f) };

Por defecto todos los widgets son responsivos: si el padre encoge, ellos encogen. Para forzar tamaño fijo usa SetNextConstraints con SizeConstraint::Fixed.

Accesibilidad

  • Tab / Shift+Tab: navega entre widgets focusables.
  • Space / Enter: activa el widget enfocado.
  • Soporte de gamepad SDL3: D-pad y A/B mapeados a navegación + activar/cancelar.
  • Anillo de foco visible automático; usa DrawAccessibilityFocusRing para widgets custom.
  • Provider UI Automation en Windows: InitUIAutomation(hwnd, ctx) expone el árbol a Narrator/NVDA.

Ventanas Demo & Inspector

El namespace FluentUI::Demo provee herramientas de desarrollo listas para enchufar. Llámalas una vez por frame desde tu root:

void Demo::ShowDemoWindow(bool* open = nullptr);
void Demo::ShowAboutWindow(bool* open = nullptr);
void Demo::ShowMetricsWindow(bool* open = nullptr);
void Demo::ShowStyleEditor(bool* open = nullptr);
void Demo::ShowItemPicker(bool* open = nullptr);
static bool showDemo    = true;
static bool showMetrics = false;
static bool showStyle   = false;
static bool showAbout   = false;

app.root([&](UIBuilder& ui) {
    ui.menuBar([&](UIBuilder& m) {
        m.menu("Ayuda", [&](UIBuilder& r) {
            r.menuItem("Demo widgets")  && (showDemo    = true);
            r.menuItem("Métricas")      && (showMetrics = true);
            r.menuItem("Style editor")  && (showStyle   = true);
            r.menuItem("Acerca de…")    && (showAbout   = true);
        });
    });

    // tu UI normal...

    if (showDemo)    Demo::ShowDemoWindow(&showDemo);
    if (showMetrics) Demo::ShowMetricsWindow(&showMetrics);
    if (showStyle)   Demo::ShowStyleEditor(&showStyle);
    if (showAbout)   Demo::ShowAboutWindow(&showAbout);

    Demo::ShowItemPicker();   // Ctrl+Shift+P activa overlay
});
  • ShowDemoWindow – cada widget en uso real con ejemplos editables.
  • ShowMetricsWindow – gráficos FPS, draw calls, batches, vertex count.
  • ShowStyleEditor – edita Style en vivo (colores, paddings, radius, tipografía).
  • ShowItemPicker – Ctrl+Shift+P: resalta el widget bajo el cursor y muestra su id-stack.
  • ShowAboutWindow – versión, build info, créditos.
